ASINs excluded from the table0 - every assigned ASIN is always represented by exactly one row.
Row grain: one row = one ASIN (REQ-02-D01 - SKU is no longer part of the report grain, the SQL grouping, or any output field). All SKU activity that used to appear as separate ASIN+SKU rows is now aggregated directly into its ASIN's single row. Vendor sales (public.vendor_sales, ASIN-only) attach directly to that same single row - duplication across former SKU rows is structurally impossible now that only one row per ASIN exists.
Image rule (REQ-02-D01): source is public.listing_data.main_image_url, joined on ref_id = ASIN, filtered to which_channel=1, market_place='UK', wrong_sku=0. When more than one valid image row exists for an ASIN, the row with the lowest listing_data.id is selected - a deterministic, run-to-run-stable technical rule, not a claim that the chosen image is "better." Business-confirmed: any valid image under the ASIN is acceptable, since all SKUs under one ASIN represent the same product. "No image available" is shown when no valid image exists; "Image loading issue" is shown when a URL exists but the browser cannot load it - these are two distinct states, never conflated.
Ordered Product Sales rule: Sales = the original product value ordered in the selected period, for the dynamic status-inclusion rule (every non-null, non-blank status except Cancelled/Canceled), source_name='AMAZON', aggregated directly at ASIN level. A refund in the same or a later month does not remove the original Sales from the month the order was placed.
Total Orders rule (ASIN-wise, REQ-02-D01, amended 2026-07-15 Section 3.2 and 2026-07-16 AMAZON-only Orders update): FBM Orders and FBA Orders = COUNT(DISTINCT order_item_info), grouped directly by (date, ASIN) at the SQL extraction layer - never grouped by SKU first and summed - and restricted to source_name='AMAZON' only (REPLACEMENT-source rows do not contribute to any Order count, per the 2026-07-16 business-rule update). Vendor Orders = public.vendor_sales.ordered_units directly (one Vendor Unit = one Vendor Order) - a direct 1:1 mapping, not a COUNT(DISTINCT order_item_info) calculation, since Vendor has no order-item key. Total Orders = FBM Orders + FBA Orders + Vendor Orders. Vendor periods (start/end date ranges, half-open [start_date, end_date)) are attributed to any selected date range they overlap, without proration - a source-data granularity limitation, not a calculation error.
Quantity removed from scope (2026-07-15, REQ-02-D01 Section 3.2): this report shows Sales and Orders only. No Quantity field (FBM/FBA/Vendor/Total/PY/CY Quantity, Quantity Change %) is calculated, displayed, or exported.
Methodology
  • Row grain: one row per ASIN (REQ-02-D01). Utharsika-assigned Amazon products only (resolved via public.user → public.ph_categories → public.ph_cate_products, DISTINCT applied, zero duplicate assignment rows verified). The table always starts from the complete assigned-ASIN master (__UAWSO_ASSIGNED_ASIN_COUNT__ ASINs) and left-joins transaction/vendor/image data - it never starts from transaction data.
  • Ordered Product Sales: Amazon UK only (source_name='AMAZON', market_place='UK'), dynamic status-inclusion rule, aggregated directly at ASIN grain and split by the fba_sales flag.
  • FBM/FBA Orders: COUNT(DISTINCT order_item_info), grouped directly by (date, ASIN), dynamic status-inclusion rule, source_name='AMAZON' only (2026-07-16 business-rule update - REPLACEMENT-source rows no longer contribute to Orders).
  • Vendor Orders (2026-07-15 amendment, REQ-02-D01 Section 3.2): public.vendor_sales.ordered_units directly - one Vendor Unit = one Vendor Order. Total Orders = FBM Orders + FBA Orders + Vendor Orders.
  • Vendor: public.vendor_sales, ordered_revenue/ordered_units, attributed once per ASIN (structurally, since there is exactly one row per ASIN).
  • Quantity is not part of this report (removed 2026-07-15, REQ-02-D01 Section 3.2) - Sales and Orders only.
  • Image: public.listing_data.main_image_url, deterministic lowest-id tie-break, business-confirmed (see Data Coverage Notes above).
  • Data included through __UAWSO_LATEST_COMPLETED_DATE__; current-day partial data excluded.
  • Trend is Sales-based only (on CY vs PY Sales): UP / DOWN / NO CHANGE. Previous=0,Current>0 → UP. Previous=0,Current=0 → NO CHANGE.
  • Achievement % measures progress against 130% of the equivalent prior-year period. Undefined ("—") when the prior-year base is zero and current > 0 — never fabricated.
  • Historical data start: __UAWSO_HISTORY_START__. Report version: __UAWSO_VERSION__.

Column Definitions

ASINThe Amazon Standard Identification Number used to group the product. One row per ASIN - all SKUs that used to belong to this ASIN are combined into this single row.
ImageShows one valid product image selected for the ASIN. If none exists, the report shows "No image available". If a valid image URL fails to load, it shows "Image loading issue".
FBM SalesOrdered product Sales for orders fulfilled by the merchant (not Amazon), for this ASIN, in the selected period.
FBM OrdersDistinct qualifying order items fulfilled by the merchant, counted once at ASIN level.
FBA SalesOrdered product Sales for orders fulfilled by Amazon, for this ASIN, in the selected period.
FBA OrdersDistinct qualifying order items fulfilled by Amazon, counted once at ASIN level.
Vendor SalesAmazon Vendor revenue for this ASIN (public.vendor_sales.ordered_revenue), attributed once per ASIN.
Vendor OrdersThe number of Vendor units ordered (public.vendor_sales.ordered_units). Under the approved business rule (2026-07-15), one Vendor unit equals one Vendor order.
Total SalesTotal qualifying product sales for all fulfilment channels under the ASIN within the selected date range, plus Vendor Sales where applicable according to the approved rule (FBM Sales + FBA Sales + Vendor Sales).
Total OrdersFBM Orders + FBA Orders + Vendor Orders.
PY SalesTotal Sales for the previous-year equivalent of the selected period.
CY SalesTotal Sales for the currently selected period.
PY OrdersTotal Orders for the previous-year equivalent of the selected period.
CY OrdersTotal Orders for the currently selected period.
Sales Change %(Current-Year Sales - Previous-Year Sales) ÷ Previous-Year Sales. Shown as "—" when the previous-year value is zero.
Order Change %(Current-Year Orders - Previous-Year Orders) ÷ Previous-Year Orders. Shown as "—" when the previous-year value is zero.
TrendUP, DOWN, or NO CHANGE, based on Current-Year Sales compared to Previous-Year Sales only.
Achievement %Current-Year Sales as a percentage of 130% of Previous-Year Sales (the sales target). Shown as "—" when the previous-year base is zero and current is greater than zero - never fabricated.
